## The Challenge

**Problem Statement:**
> "Every night, millions spend up to 45 minutes deciding what to watch â€” billions of hours lost every day. Not from lack of content, but from fragmentation."

**Root Causes:**
1. Content spread across 200+ streaming services globally
2. No unified search across platforms
3. Availability varies by region and changes daily
4. Metadata quality inconsistent across sources
5. No standardized way for AI agents to discover content

**Solution Vision:**
An AI-native discovery platform where users can say:
> "Find me a rom-com streaming tonight"

...and get an instant, verified answer with direct links.

## ARW (Agent-Ready Web) Specification

ARW is a specification for making web content efficiently consumable by AI agents.

### Core Concepts

| Concept | Description |
|---------|-------------|
| **llms.txt** | Discovery file at site root (like robots.txt for AI) |
| **Machine Views** | Structured markdown versions of pages (`.llm.md`) |
| **arw-manifest.json** | JSON manifest in `.well-known/` directory |
| **Actions** | Defined API endpoints agents can invoke |
| **Policies** | Training/inference permissions, rate limits |

### Benefits

- **85% token reduction** vs HTML scraping
- **10x faster discovery** via structured manifests
- **OAuth-enforced actions** for safe agent transactions
- **Full observability** via AI-* headers

### ARW Manifest Structure

```json
{
  "version": "0.1",
  "profile": "ARW-1",
  "site": {
    "name": "AI Media Discovery",
    "description": "Natural language media search"
  },
  "content": [
    {
      "url": "/search",
      "machine_view": "/llms/search.llm.md",
      "purpose": "search",
      "priority": "high"
    }
  ],
  "actions": [
    {
      "id": "semantic_search",
      "endpoint": "/api/search",
      "method": "POST",
      "schema": { ... }
    }
  ],
  "policies": {
    "training": { "allowed": false },
    "inference": { "allowed": true },
    "rate_limits": { "authenticated": "1000/min" }
  }
}
```

### llms.txt Format

```yaml
version: 0.1
profile: ARW-1

site:
  name: 'AI Media Discovery'
  description: 'Discover content via natural language'

content:
  - url: /search
    machine_view: /llms/search.llm.md
    purpose: search

actions:
  - id: semantic_search
    endpoint: /api/search
    method: POST
```

## Data Curation Architecture

### The Core Challenge

No single source has everything:
- TMDB has posters but missing older content
- IMDb has breadth but no images
- Regional sources have local content but limited global reach
- Streaming availability changes daily

### Entity Resolution Strategy

Use **Wikidata QID as canonical identifier** with mappings to:
- IMDB ID (tconst)
- TMDB ID
- TVDB ID
- EIDR
- Regional IDs (Douban, AniDB, etc.)

## Proposed Solution

### Project: Agentic Media Data Curator

**Goal:** Build an AI-powered data curation pipeline that:
1. Ingests from 10+ metadata sources
2. Resolves entities into a unified schema
3. Enriches with streaming availability
4. Exposes via ARW-compliant API
5. Updates continuously via agentic workflows

### Key Deliverables

1. **Data Ingestion Adapters**
   - TMDB adapter (primary metadata)
   - IMDb dataset importer
   - Wikidata SPARQL connector
   - Streaming availability API client

2. **Entity Resolution Engine**
   - Wikidata ID linker
   - Fuzzy matching algorithm
   - Confidence scoring

3. **Unified Content Schema**
   - Canonical metadata model
   - Multi-language support
   - Cross-reference ID mapping

4. **ARW API Layer**
   - REST endpoints for search/discover
   - llms.txt and arw-manifest.json
   - Machine views for content pages

5. **Agentic Curation Workflows**
   - Scout/Match/Enrich/Validate agents
   - Scheduled freshness checks
   - Quality monitoring dashboard

- **Coverage:** 100K+ titles with unified metadata
- **Quality:** 95%+ entity resolution accuracy
- **Freshness:** <24hr streaming availability updates
- **Performance:** <500ms API response time
